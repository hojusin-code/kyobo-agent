교보문고 AI 발주 Agent — 사내 서버 배포 가이드 (IT팀 전달용)
================================================================

■ 파일 구성
  - index.html   : 메인 화면 (프론트엔드, 수정 불필요)
  - server.js    : 서버 실행 파일 (Node.js 전용, 외부 라이브러리 설치 불필요)

이 두 파일만 같은 폴더에 두면 됩니다.


================================================================
방법 A (권장) — 서버에 Node.js가 설치되어 있는 경우
================================================================

1. 두 파일(index.html, server.js)을 서버의 원하는 폴더에 업로드
2. 해당 폴더에서 아래 명령어 실행

     node server.js

3. 기본 포트는 3000번입니다. 접속 주소: http://서버주소:3000
   포트를 바꾸려면 실행 전에 PORT 환경변수를 지정하면 됩니다.
   (예: Windows  set PORT=8080 && node server.js)
4. 서버가 항상 켜져 있어야 하므로, 필요하면 PM2 같은 프로세스
   매니저나 Windows 서비스 등록으로 상시 실행되도록 설정해 주세요.

Node.js가 설치되어 있는지 확인하려면: node -v


================================================================
방법 B — Node.js를 사용할 수 없는 환경인 경우
================================================================

server.js를 실행할 수 없다면, 아래 API 2개만 사내에서 쓰는
언어(PHP·ASP.NET·Java 등)로 똑같이 만들어 주시면 됩니다.
로직은 "외부 API를 대신 호출해서 결과를 그대로 돌려주는" 역할뿐입니다.

--------------------------------------------------------------
[API 1] GET /api/aladin?isbn={ISBN}
--------------------------------------------------------------
아래 알라딘 API를 서버에서 호출:

  https://www.aladin.co.kr/ttb/api/ItemLookUp.aspx
    ?ttbkey=ttbhojusin1324001
    &itemIdType=ISBN13
    &ItemId={ISBN}
    &output=js
    &Version=20131101
    &OptResult=bestSellerRank,reviewList
    &Cover=Big

받은 응답(JSON)의 item 배열 중 첫 번째 항목만 꺼내서 아래 형태로 반환:

  { "item": {
      "title": "...", "author": "...", "publisher": "...",
      "pubDate": "...", "cover": "...", "salesPoint": 0,
      "bestSellerRank": null, "customerReviewRank": 0,
      "categoryName": "...", "link": "..."
  }}

item이 없으면 { "item": null } 반환.

--------------------------------------------------------------
[API 2] GET /api/search?isbn={ISBN}
--------------------------------------------------------------
아래 네이버 검색 API를 서버에서 호출 (헤더 필요):

  https://openapi.naver.com/v1/search/book_adv.json?d_isbn={ISBN}&display=1
  헤더: X-Naver-Client-Id: LQiBBpg9tPtr0CU8q6EK
        X-Naver-Client-Secret: aEVIAXnb7M

받은 결과의 items[0]을 item으로 사용하고, item.title에서 태그·괄호를
제거한 뒤 아래 API로 재검색해서 검색 결과 총 개수(searchCount)를 구함:

  https://openapi.naver.com/v1/search/book.json?query={제목}&display=5
  (같은 헤더 사용)

최종 응답 형태:

  { "item": {...네이버 도서 정보...}, "searchCount": 123 }

--------------------------------------------------------------

※ 이 2개 API가 없어도 프로그램의 핵심 기능(신간 목록, AI 발주량 계산,
   발주 근거 표시, 최종 발주 확정)은 정상 작동합니다. 다만 알라딘
   판매지수·표지 이미지·네이버 관심도 표시 기능만 빠지게 됩니다.
   index.html은 이 API가 없으면 자동으로 해당 기능을 조용히 건너뜁니다
   (에러 화면 뜨지 않음).


================================================================
보안 주의사항
================================================================

- 위 API 키(ttbkey, X-Naver-Client-Id, X-Naver-Client-Secret)는
  반드시 서버 코드 안에만 위치해야 하며, 브라우저(프론트엔드)로
  전달되거나 노출되면 안 됩니다.
- index.html 파일 안에는 이 키들이 전혀 포함되어 있지 않습니다.
  (서버 쪽 파일에만 존재)
