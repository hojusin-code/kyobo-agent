## 📋 발표 순서 (말하는 순서)

1. **비전** — 최종적으로 만들고 싶은 시스템이 무엇인지 먼저 보여준다 (1번)
2. **데모** — 지금 그 비전의 어디까지 구현됐는지 직접 화면으로 보여준다 (2~5번)
3. **실증** — 실제 회사 데이터를 넣어봤더니 어디가 비는지 증명한다 (6번) ← *이게 핵심 설득 포인트*
4. **요청** — 그래서 회사가 무엇을 지원해줘야 하는지, 왜 지금 필요한지 말한다 (7번)

> 핵심 메시지 한 줄: **"방향과 로직은 검증됐다. 실제 데이터만 연결되면 1개월 안에 현장에서 쓸 수 있다."**

## 1. 비전 — 최종 목표

> 신간 입고 당일, 담당자가 ISBN 하나만 입력하면
> **실시간 판매 데이터 기반으로 AI가 발주량과 근거를 즉시 제시**하고,
> 확정 즉시 기존 발주 시스템으로 바로 넘어가는 시스템.

### 목표 기한
**1개월 내** 목업 데이터를 실제 회사 DB 연동으로 전환하여, 대구점 현장에서 시범 사용 가능한 수준으로 완성.

### 왜 필요한가 (문제 정의)
신간 도서 입고 당일, 담당자가 수십 종의 책을 빠르게 검토하고 적정 발주량을 결정해야 한다.
기존에는 직접 판매 데이터를 찾아보며 경험에 의존해 결정 — **시간 소모 크고, 과재고·품절 리스크 존재**.

## 3. AI 발주 계산 공식 (로직은 이미 완성됨)

```
기준수량 = 직전권 7일 판매 × 60% + 시리즈 평균 7일 × 40%
          (시리즈 1권뿐이면 직전권 100%)

예측 7일 판매량 = 기준수량
               × 시리즈 추세 보정 (×0.92 ~ ×1.12)
               × 시장 성장률 보정 (×0.97 ~ ×1.06)

AI 추천 발주량 = 예측 7일 판매량 − 현재 재고 − 입고 예정
```

**핵심 설계 의도:** 직전 2주 판매량을 7일 기준으로 쪼개어 사용 → 뒤늦게 판매되는 신간 특성 반영, 과재고 방지

|---|---|---|

| 트렌드 지수 | 20% | 실시간 검색 관심도·SNS 반응 |

## 6. 실데이터로 검증한 결과 — "방향이 맞다"는 증거

회사로부터 받은 실제 신간목록 29종(`신간목록_20260625.xlsx`)과 판매 데이터(`발주데이터_정확한판매수량.xlsx`, 1,004종)를
**목업이 아닌 실제 화면에 직접 연동**해서 테스트함. 결과:

- ✅ **로직·화면·계산식은 정상 작동** — ISBN 입력 → 발주 분석 → 확정까지 끊김 없이 동작
- ⚠️ **단, 실제 데이터에 구멍이 있는 항목은 화면에 "데이터없음"으로 정직하게 표시됨** (가짜로 채우지 않음)

| 받은 데이터 | 계산식이 필요로 하는 데이터 | 현재 결과 |
|---|---|---|
| 직전권 **누적** 판매수량 | 직전권 **출간일 기준 7일/14일** 판매량 | 7일/14일 카드 → 0권 (데이터없음) |
| 입고예정일만 있음 | 현재 재고 + 입고예정 **수량** | 재고/입고 → 0권 (미제공) |
| 도서 마스터(제목/저자/카테고리)만 있음 | 저자·시리즈·시장·트렌드 점수 | 이슈점수 → "데이터없음" 배지, 우선검토 0종 |

**결론:** 30종을 직접 화면에 태워본 결과 시스템 자체는 실데이터를 받을 준비가 끝났고, **막혀있는 건 데이터뿐**이라는 게 명확히 증명됨. 이게 7번 요청사항의 근거.

## 7. 회사에 필요한 것 (지원 요청)

★★ 사번 로그인 시 해당 지점으로 자동 세팅 필요 (현재 통유시스템 로그인 기준)

### 🔴 즉시 필요 — 1개월 목표의 전제조건
1. **실제 판매 데이터 API 연동**
   - 현재: 직전권 **누적**판매수량만 확인 가능 (위 6번 테스트로 확인)
   - 필요: ISBN별 **출간일 기준 7일/14일 판매량**, 현재 재고, 입고예정 **수량**

2. **신간 입고 데이터 자동 연동**
   - 현재: 날짜 필터로 수동 조회, 입고예정일만 제공됨
   - 필요: 당일 입고 리스트 + 수량 자동 푸시 (API 또는 일일 파일 연동)

### 🟡 중기 필요
3. **사내 배포 환경** — Vercel 개인 계정 → 사내 인트라넷/접근 제어 서버
4. **발주 확정 시스템 연동** — 클립보드 복사 → 발주 시스템 직접 전송 API

### 🟢 장기 검토
5. **실제 트렌드 데이터 연동** (네이버 검색량, SNS 언급량, yes24·알라딘 등)
6. **멀티 사용자 지원** (여러 담당자 동시 사용, 분야별 뷰)
7. **발주 이력 관리** (과거 추천 vs 실제 발주 결과 비교)
